### [JSDoc is TypeScript](https://culi.bearblog.dev/jsdoc-is-typescript/) • 92pts 127c
[HN discussion](https://news.ycombinator.com/item?id=46266102)
TLDR: 기사는 TypeScript 검사를 사용하는 JSDoc이 빌드 단계 없이 타입 안전성을 제공한다고 주장한다. JSDoc 주석이 있는 JavaScript를 작성하고, TypeScript가 검증하고, 네이티브 JS를 배포한다. 트랜스파일에 알레르기가 있는 사람들을 위한 양쪽 세계의 최고.
Take: 댓글이 정점 HN이다. 한 사람이 타입이 "머릿속에 작성"되어 있고 그게 "매우 휘발성"이라는 경력 지혜를 공유. 또 다른 사람은 커뮤니티가 Closure Compiler보다 네이티브 TS를 선택한 것을 한탄. 진짜 요점은 모든 JS 프로젝트가 결국 같은 타입 안전성 바퀴를 재발명한다는 것, 다만 다른 모양의 살만 있을 뿐.
댓글:
- "타입은 작성하든 안 하든 존재한다. 작성하지 않으면 머릿속에 작성되어 있다. 머리는 매우 휘발성이다." -Waterluvian
Tags: #javascript #typescript #jsdoc #webdev

### [Shai-Hulud compromised a dev machine and raided GitHub org access: a post-mortem](https://trigger.dev/blog/shai-hulud-postmortem) • 186pts 111c
[HN discussion](https://news.ycombinator.com/item?id=46262021)
TLDR: Trigger.dev가 손상된 npm 패키지를 통한 공급망 공격을 당했다. Shai-Hulud (맞다, Dune 샌드웜)라는 공격자가 npm post-install 스크립트를 사용해 개발자의 머신을 손상시킨 후 GitHub org 자격증명과 AWS 비밀에 접근했다. 그들은 post-install 스크립트를 기본적으로 차단할 것을 요청하고 있다.
Take: 저자의 "npm install 실행은 과실이 아니다"라는 방어는 제대로 구워졌다. 패키지 관리자가 설치 시 임의 코드 실행을 허용할 때, 전체 npm 생태계를 신뢰하는 것이 과실이다. post-install 스크립트 차단이 고통스럽지만 필요하다는 pnpm 메인테이너의 인용은 고급 와인처럼 숙성되었다.
댓글:
- "아니, 당신의 보안 실패는 감독 없이 제3자가 임의 코드를 제품에 푸시할 수 있게 하는 패키지 관리자를 사용한다는 것이다." -snickerbockers
Tags: #security #npm #supplychain #javascript

### [Hashcards: A plain-text spaced repetition system](https://borretti.me/article/hashcards-plain-text-spaced-repetition) • 238pts 101c
[HN discussion](https://news.ycombinator.com/item?id=46264492)
TLDR: Hashcard는 카드가 콘텐츠 해시로 식별되는 CLI 플래시카드 시스템이다. FSRS 알고리즘 (SM-2보다 새로운 핫한 것) 사용, 모든 것을 평문으로 저장, 모든 텍스트 에디터에서 카드 편집 가능. Anki가 너무 무겁다고 생각하는 사람들을 대상으로 한다.
Take: 댓글에서 누군가 아내의 치과 상황으로 행동 수정에 간격 반복을 사용하는 것이 오늘 내가 읽은 가장 HN스러운 것이다. 또한 콘텐츠 해싱이 카드 편집이 진행 상황을 재설정한다는 공정한 비판. 단순성과 기능성 사이의 영원한 트레이드오프.
댓글:
- "간격 반복의 진짜 힘은 이런 플래시카드 애플리케이션에 있지 않다고 생각한다. 행동 수정에 있다." -btilly
