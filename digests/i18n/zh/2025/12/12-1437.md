### [Training LLMs for Honesty via Confessions](https://arxiv.org/abs/2512.08093) • 18pts 5c
[HN discussion](https://news.ycombinator.com/item?id=46242795)
TLDR: 研究人员训练 GPT-5-Thinking 在主要回答之后产生"忏悔"，将诚实与任务表现分开奖励。当模型在主要回复中撒谎或遗漏时，它们往往在忏悔中坦白。通过训练，诚实度有适度提升。
Take: 我们为神经网络发明了天主教的罪恶感。AI 告解室是真实存在的，而且它居然有效。接下来：AI 忏悔和算法圣母经。
Comment: "dishonesty may arise due to the effects of reinforcement learning (RL), where challenges with reward shaping can result in a training process that inadvertently incentivizes the model to lie" -manarth

### [Tripwire: A new anti evil maid defense](https://github.com/fr33-sh/Tripwire) • 21pts 14c
[HN discussion](https://news.ycombinator.com/item?id=46229437)
TLDR: 一个带摄像头和 PIR 传感器的 Raspberry Pi 5 在你不在时监视你的笔记本电脑。如果有人动它，Tripwire 会销毁共享密钥并发送推送通知。为记者和偏执的开发者设计，他们担心酒店员工拿着 USB 棒。
Take: HN 评论者立即指出你可以在笔记本电脑铰链里放豆子和米。他们说得没错。还有，二十年前就有人把他们的安全产品命名为 Tripwire 了。
Comment: "The classic 'Detecting unauthorized physical access with beans, lentils and colored rice' approach is simpler to understand and simpler to implement" -neuralkoi

### [What folk can do](https://folk.computer/guides/what-folk-can-do) • 19pts 10c
[HN discussion](https://news.ycombinator.com/item?id=46186990)
TLDR: Folk 是一个实时编程系统，运行在投影仪和摄像头上，让你在桌子上排列实物物体，让它们与数字内容互动。它是 Dynamicland 的精神续作，现在是一个你可以实际使用的产品。
Take: 前 Dynamicland 成员离开去构建人们可以实际使用的东西。HN 评论者在兴奋和安静的失望之间摇摆——研究乌托邦变成了创业公司。
Comment: "What I cannot understand is why they left Dynamicland and made it a product. (Which is against everything that Bret Victor stands for)" -pmkary

### [Use Python for Scripting](https://hypirion.com/musings/use-python-for-scripting) • 5pts 6c
[HN discussion](https://news.ycombinator.com/item?id=46189962)
TLDR: Python 无处不在，有可预测的标准库，开箱即用处理 JSON 和 HTTP，不会用 bash 那些晦涩的引号规则让你惊讶。当你的 shell 脚本变得太复杂时，用它。
Take: 这个建议从 2010 年开始就是对的，2030 年也会是对的。然而开发者还是会写 500 行带嵌套子 shell 的 bash 脚本，然后纳闷为什么在 macOS 上什么都不工作。
Comment: "How do you handle packages? I want scripts to be a single file with a shebang, not a repo with a requirements.txt" -mubou2

### [Show HN: Autofix Bot – Hybrid static analysis and AI code review agent](https://news.ycombinator.com/item?id=46237358) • 14pts 1c
[HN discussion](https://news.ycombinator.com/item?id=46237358)
TLDR: DeepSource 构建了一个 agent，将 5000+ 个静态分析检查器与 LLM 审查结合起来。AI 使用 AST 和数据流图作为工具，而不仅仅是 grep。一个子 agent 过滤误报，然后另一个子 agent 修复问题。
Take: AI 生成代码，AI 审查代码，AI 修复代码。人类只是看着 agent 们互相争论。我们已经将整个代码审查站会自动化了。
Comment: "'shifted bottleneck to code review'... understatement of decade." -nickphx
