### [JSDoc 就是 TypeScript](https://culi.bearblog.dev/jsdoc-is-typescript/) • 92pts 127c
[HN discussion](https://news.ycombinator.com/item?id=46266102)
TLDR：文章认为带 TypeScript 检查的 JSDoc 可以让你在没有构建步骤的情况下获得类型安全。你用 JSDoc 注解写 JavaScript，TypeScript 验证它们，然后你发布原生 JS。对那些对转译过敏的人来说是两全其美。
Take：评论是 HN 的巅峰。一个人分享关于类型"写在你脑子里"而那"非常易失"的职业智慧。另一个人哀叹社区选择了原生 TS 而不是 Closure Compiler。真正的要点是每个 JS 项目最终都重新发明同样的类型安全轮子，只是用不同形状的辐条。
Comment："类型存在，无论你是否写下来。如果它们没被写下来，它们就写在你脑子里。你的大脑非常易失。" -Waterluvian

### [Shai-Hulud 攻陷了开发机器并洗劫了 GitHub org 访问权限：事后报告](https://trigger.dev/blog/shai-hulud-postmortem) • 186pts 111c
[HN discussion](https://news.ycombinator.com/item?id=46262021)
TLDR：Trigger.dev 通过一个被攻陷的 npm 包遭受供应链攻击。一个名叫 Shai-Hulud（是的，沙丘里的沙虫）的攻击者使用 npm post-install 脚本攻陷了开发者的机器，然后访问了 GitHub org 凭据和 AWS 秘密。他们呼吁默认阻止 post-install 脚本。
Take：作者为"运行 npm install 不是疏忽"的辩护被恰当地烤了。当你的包管理器允许在安装时执行任意代码，信任整个 npm 生态系统才是疏忽。pnpm 维护者关于阻止 post-install 脚本痛苦但必要的引用已经像陈年美酒一样。
Comment："不，你的安全失误是你使用的包管理器允许第三方在没有监督的情况下将任意代码推入你的产品。" -snickerbockers

### [Hashcards：纯文本间隔重复系统](https://borretti.me/article/hashcards-plain-text-spaced-repetition) • 238pts 101c
[HN discussion](https://news.ycombinator.com/item?id=46264492)
TLDR：Hashcards 是一个 CLI 闪卡系统，其中卡片由内容哈希标识。使用 FSRS 算法（比 SM-2 更新的热点），将所有内容存储为纯文本，让你在任何文本编辑器中编辑卡片。针对觉得 Anki 太重的人。
Take：评论中有人用间隔重复对他妻子的牙医情况进行行为矫正，这是我今天读到的最 HN 的东西。另外，公正的批评是内容哈希意味着编辑卡片会重置你的进度。简单性和功能性之间的永恒权衡。
Comment："我认为间隔重复的真正力量不在于像这样的闪卡应用。它在于行为矫正。" -btilly
