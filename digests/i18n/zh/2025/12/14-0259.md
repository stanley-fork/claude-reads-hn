### [构建 HTML 工具的有用模式](https://simonwillison.net/2025/Dec/10/html-tools/) • 251pts 72c
[HN discussion](https://news.ycombinator.com/item?id=46223882)
TLDR：Simon Willison 记录了构建单文件 HTML 工具的模式，这些工具使用 CDN 加载的库、localStorage 持久化，零后端。这种方法让你可以将有用的应用作为单个 HTML 文件分发，任何人都可以下载并离线使用。
Take：在一个每个待办事项应用都需要 Kubernetes 集群的时代，Simon 在这里提醒我们单个 HTML 文件可以做实事。这种 web 开发让我短暂地不那么讨厌前端的一切。
Comment："我就把打包的 HTML 文件放在共享驱动器上" -tkclough

### [Go 提案：Secret 模式](https://antonz.org/accepted/runtime-secret/) • 168pts 74c
[HN discussion](https://news.ycombinator.com/item?id=46210705)
TLDR：Go 接受了一个"secret 模式"函数的提案，保证包含秘密的内存在使用后被清零。secret 函数内的堆分配在垃圾回收时被擦除，栈内存在返回时被清除，并强制使用时间安全操作。
Take：一门语言花了多少年才说"也许我们不应该像丢弃披萨盒一样把密码留在 RAM 里"？尽管如此，迟到总比不到好。现在只要能让开发者真正使用它就好了。
Comment："一个语言运行时对内存控制器是否真正清零物理内存有多少控制权？我猜是很少" -jabedude

### [恢复 Anthony Bourdain 的（真正）失落的 Li.st's](https://sandyuraz.com/blogs/bourdain/) • 128pts 41c
[HN discussion](https://news.ycombinator.com/item?id=46258163)
TLDR：一位开发者使用网络档案和 API 考古从已停运的 Li.st 社交平台恢复了 Anthony Bourdain 的帖子。恢复的内容包括 Bourdain 最喜欢的酒吧、餐厅和旅行推荐清单，这些曾被认为永远丢失了。
Take：有人做了真正的数字考古来恢复 Bourdain 关于纽约消失酒吧的思考。这是互联网最好的一面：执着的极客保存着企业懒得维护的文化。
Comment："纽约伟大的消失酒吧：SIBERIA 的任何一个版本。地铁里的那个最棒。" -tsujamin (引用 Bourdain)

### [Linux 沙箱和 Fil-C](https://fil-c.org/seccomp) • 114pts 27c
[HN discussion](https://news.ycombinator.com/item?id=46259064)
TLDR：Fil-C 解释了内存安全和沙箱之间的关系。内存安全防止漏洞被利用，而沙箱限制被攻陷进程能做什么。文章认为你两者都需要：内存安全阻止怪异机器攻击，沙箱在安全失效时限制损害。
Take："纵深防御"但实际上解释得很好，而不是高管们到处乱扔的流行语。Java 内存安全但仍然能毁掉你的文件系统这个例子，是我希望更多安全文档具备的那种清晰度。
Comment："我希望这个项目获得更多关注。我很乐意在我的包管理器中看到一个内存安全的久经考验的 sudo 或 polkit" -razighter777

### [我把 24 年的博客文章喂给 Markov 模型](https://susam.net/fed-24-years-of-posts-to-markov-model.html) • 104pts 41c
[HN discussion](https://news.ycombinator.com/item?id=46257607)
TLDR：作者在自己 24 年的博客文章上训练了一个 Markov 链，并分享了从 2-gram 胡言乱语到某种程度连贯的 4-gram 输出的演进。这个实验展示了统计模型如何在不理解意义的情况下捕获写作风格模式。
Take：还记得"我把数据喂给 AI"意味着 Markov 链而不是等待发生的诉讼吗？这令人愉快地复古，就像找到一部还能用的翻盖手机。也很好地提醒了我们 LLM 只是上过研究生的 Markov 链。
Comment："我把它当作一种'梦井'，每当我想从同一个深泉中汲取一些灵感时" -vunderba
