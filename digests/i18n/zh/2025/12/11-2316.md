### [React Server Components 中的拒绝服务和源代码暴露](https://react.dev/blog/2025/12/11/denial-of-service-and-source-code-exposure-in-react-server-components) • 133pts 55c
[HN discussion](https://news.ycombinator.com/item?id=46236924)
TLDR: React Server Components 遭遇两个严重 CVE——导致 DoS 攻击的无限循环和服务器函数的潜在源代码暴露（包括硬编码的机密）。补丁立即可用。
Take: Server Components 本应让 React 开发更容易，但反而创造了一类新的安全陷阱。也许我们应该坚持那种"与服务器详细对话"的模式。
Comment: "React Server Components 总是让我感到不舒服，因为它们让人很难看一段 JavaScript 代码并推断出哪些部分将在客户端运行，哪些部分在服务器上运行" -simonw
