[HN discussion](https://news.ycombinator.com/item?id=46243543)
TLDR: Tor 通过 Arti 项目用 Rust 重写他们的 C 代码库。1.8.0 版本带来了电路超时改进和实验性的 onion 服务密钥迁移。主要目标：消除困扰当前实现的缓冲区溢出和释放后使用的错误。

### [从文本到 token：tokenization 管道如何工作](https://www.paradedb.com/blog/when-tokenization-becomes-token) • 23pts 1c
[HN discussion](https://news.ycombinator.com/item?id=46232003)
TLDR: 深入探讨传统搜索 tokenization：字符过滤、小写规范化、停用词移除和词干提取。涵盖"jumped"和"jumping"如何都变成"jump"以进行匹配。值得注意的是与 LLM tokenization 不同，后者有完全不同的目标。
Take: 终于有一篇关于 tokenization 的文章不是试图向你推销另一个 LLM 包装器。搜索引擎安静的不起眼的管道工作得不到 Twitter 讨论串或 VC 资金，但正是它让互联网真正可查找。
Comment: "值得注意的是传统搜索的 tokenization。LLM 使用非常不同的 tokenization，有非常不同的目标" -wongarsu

### [使用生成列以全索引速度处理 SQLite JSON](https://www.dbpro.app/blog/sqlite-json-virtual-columns-indexing) • 4pts 1c
[HN discussion](https://news.ycombinator.com/item?id=46243904)
TLDR: 在 SQLite 中存储 JSON blob，然后用 json_extract() 创建虚拟生成列并对其索引。你可以获得文档存储的模式灵活性和关系查询性能。无需 ETL，无需数据迁移，只需根据需要添加列。
Take: SQLite 继续其安静的世界统治。当所有人争论为哪个云数据库支付过高费用时，SQLite 只是不断添加功能，让 90% 的数据库用例在单机上变得简单可解决。这场复兴是真实的。
Comment: "我受到 bambax 在 2023 年 HN 帖子上的评论启发写了这篇博客文章" -upmostly

### [Fedora：长期数字保存的开源仓库](https://fedorarepository.org/) • 7pts 2c
[HN discussion](https://news.ycombinator.com/item?id=46243883)
TLDR: 不是 Linux 发行版。FEDORA 代表 Flexible Extensible Digital Object Repository Architecture，康奈尔大学的一个项目，用于长期保存数字资产。图书馆和档案馆用它来确保研究和文化遗产比它们的原始格式存活更久。
Take: 与 Red Hat 发行版的命名冲突意味着这个有用的工具可能失去了 50% 的潜在受众因为点击混淆。但数字保存是那些无聊的基本问题之一，没人思考，直到他们最喜欢的网站消失或博物馆失去几十年的数字艺术。
Comment: "为避免误解，这个仓库是关于康奈尔大学的一个名为 Flexible Extensible Digital Object Repository Architecture (FEDORA) 的项目，而不是 Red Hat 的。" -fodmap
